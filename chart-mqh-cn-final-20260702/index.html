Chart.mqh 中文注释版说明

本页面用于说明本次汉化的 Chart.mqh 源码文件。汉化原则是保留类名、函数名、参数名和执行逻辑，只翻译注释与说明，避免破坏 MQL 编译兼容性。

汉化源码文件名：Chart_Chinese_Commented.mqh。该文件已在本次聊天中单独交付。

一、先说结论：它不是指标

这份源码不是交易指标，也不是买卖信号工具。源码里没有 OnCalculate()、没有指标缓冲区、没有均线/RSI/MACD 等行情公式，也没有多空箭头、开仓条件、止盈止损逻辑。

它的准确定位是：MetaTrader 图表对象封装类 / 图表控制基础库。

二、核心结构

CChart 类内部围绕 m_chart_id 这个图表标识符工作。先绑定或打开图表，再对指定图表执行读取、设置、导航、截图、指标管理和参数保存/读取。

模块代表函数说明
图表绑定Attach()、Open()、FirstChart()、NextChart()确定对象当前控制哪一个图表。
属性读取Mode()、Scale()、ColorBackground()读取图表显示模式、缩放、颜色、尺寸等状态。
属性设置ShowGrid()、ColorGrid()、AutoScroll()修改图表外观和交互状态。
指标管理IndicatorAdd()、IndicatorDelete()、IndicatorsTotal()把已有指标句柄挂到图表、删除指标、查询指标数量。
持久化Save()、Load()保存和恢复图表配置。

三、功能介绍

它封装了图表显示模式、前景显示、右侧留白、自动滚动、缩放、固定价格范围、OHLC 显示、Bid/Ask/Last 线显示、周期分隔线、网格、成交量、对象说明、日期坐标轴、价格坐标轴、图表颜色、事件开关、窗口尺寸、价格上下限、截图、模板应用、图表导航、指标挂载和文件保存读取。

四、逻辑性评价

工程逻辑是成立的。它把底层 ChartGetInteger、ChartSetInteger、ChartGetDouble、ChartSetDouble、ChartGetString、ChartSetString 等 API 包装成类方法，让 EA、指标或脚本调用更统一。

但它没有交易逻辑，对盈利没有直接贡献。它只能作为 EA 面板、图表模板、多图表控制、自动截图、指标加载管理等模块的底层工具。

五、注意点

  • MQL4 分支里 ChartIndicatorAdd() 直接返回 false，说明 MQL4 环境下这部分通过句柄添加指标的能力不可用。
  • 析构函数 ~CChart() 会在 m_chart_id!=-1 时调用 Close()，如果把当前图表绑定给临时对象，生命周期结束时可能关闭图表。
  • Save() 使用 FileWriteDouble() 写入部分属性，但 Load() 中对应位置使用 FileReadDatetime() 读取，属于疑似不一致点。汉化版没有擅自修改逻辑。

六、对外介绍文案

Chart.mqh 中文注释版 是一个面向 MetaTrader 开发者的图表控制基础库。它通过 CChart 类封装图表显示、缩放、颜色、坐标轴、事件开关、窗口信息、指标挂载、截图和参数持久化等功能，让 EA、指标和脚本开发中的图表操作更清晰、更模块化。

它不是独立交易指标，也不提供买卖信号，更适合作为复杂 EA 面板、图表模板管理、多窗口控制、自动截图工具、指标加载管理模块的底层组件。
