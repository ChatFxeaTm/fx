MQL4 / MQL5中文注释版图表管理类

Chart.mqh 中文注释版说明

这份文件不是交易指标源码，而是一个 图表对象封装类：通过 CChart 类统一封装 MetaTrader 图表属性读取、图表属性设置、图表窗口信息获取、指标挂载/删除，以及图表参数保存/读取。

汉化原则：保留类名、函数名、参数名和执行逻辑，只翻译注释与说明，避免破坏编译兼容性。

一、先把性质说清楚

这不是指标。源码里没有 OnCalculate()、没有指标缓冲区 SetIndexBuffer()、没有均线/RSI/MACD 等信号计算，也没有买卖箭头、开仓条件、止盈止损逻辑。它的作用是给 EA、指标或脚本提供一套“操作图表”的工具类。

所以，如果你要做商业化介绍，不能把它包装成“能判断行情方向的指标”。它更准确的定位是：MetaTrader 图表控制基础库。

二、核心逻辑结构

CChart 类内部只保存一个关键成员：

long m_chart_id; // 图表标识符

所有方法都围绕这个图表 ID 展开：先绑定或打开图表，然后对这个图表进行读取、设置、导航、截图、指标管理和参数保存。

逻辑阶段代表函数作用说明
绑定图表Attach()、Open()、FirstChart()、NextChart()确定当前对象要操作哪一个图表。
读取属性Mode()、Scale()、ColorBackground() 等从指定图表读取显示模式、缩放、颜色、窗口尺寸等状态。
设置属性Mode(mode)、ShowGrid(show)、ColorGrid(color) 等修改图表显示效果和交互状态。
图表导航Navigate()控制图表跳转到指定位置，例如最新K线、历史位置等。
指标管理IndicatorAdd()、IndicatorDelete()、IndicatorsTotal()在图表子窗口中添加、删除或查询指标。
持久化Save()、Load()把图表配置写入文件，或从文件恢复配置。
释放资源Close()、析构函数 ~CChart()关闭已绑定的图表并重置 ID。

三、功能模块详解

1. 图表显示模式

Mode() 用于读取或设置图表显示方式，比如柱状图、蜡烛图、折线图。它本身不分析行情，只改变图表外观。

2. 缩放与滚动

Scale() 控制缩放级别，代码中把范围限制在 0~32；ShiftSize() 把右侧留白限制在 10~50，避免传入异常参数导致图表显示失控。

3. 显示开关

包括 OHLC、Bid 线、Ask 线、Last 线、周期分隔线、网格、成交量、对象说明、日期坐标轴、价格坐标轴等。适合做模板化图表界面。

4. 颜色控制

封装背景色、前景色、网格色、阳线色、阴线色、成交量色、Bid/Ask/Last 线颜色、止损止盈水平线颜色等设置函数。

5. 事件开关

EventObjectCreate()、EventObjectDelete()、EventMouseMove() 用于控制图表事件是否被触发。开发可拖动面板、按钮、图形对象时会用到。

6. 只读窗口信息

读取可见K线数量、窗口数量、窗口像素尺寸、价格上下限、子窗口偏移等。它适合做图表面板布局、自动截图、界面自适应。

7. 指标挂载管理

IndicatorAdd() 可以把已有指标句柄添加到指定图表窗口；IndicatorDelete() 根据名称删除指标；IndicatorsTotal() 查询子窗口指标数量。

8. 参数保存/读取

Save() 和 Load() 用文件方式保存图表部分配置，便于恢复图表模板状态。它是工程功能，不是交易策略。

四、逻辑性评价

工程逻辑是成立的。它的思路是把底层 ChartGetInteger、ChartSetInteger、ChartGetDouble、ChartSetDouble、ChartGetString、ChartSetString 等函数包成类方法，让外部代码调用更统一、更清楚。

但它没有任何市场判断能力。严格说，它对交易盈利没有直接贡献，只能提高 EA/指标开发时的图表控制能力。

评价维度结论
交易信号逻辑没有。不能判断多空，不能给出入场点。
指标计算逻辑没有。没有指标缓冲区，没有数学公式计算。
图表控制逻辑完整。覆盖显示、颜色、事件、窗口信息、指标管理、文件保存读取。
商业化价值不能单独当指标卖；可以作为 EA 面板、图表模板、辅助工具库的一部分。
适合使用场景EA 图表界面、自动截图、模板管理、多图表控制、指标加载管理。

五、需要注意的代码点

  • MQL4 兼容处理：在 __MQL4__ 下，ChartIndicatorAdd() 被定义为直接返回 false，说明 MQL4 环境里这部分“通过句柄添加指标”的能力不可用。
  • 析构函数会关闭图表：~CChart() 中如果 m_chart_id!=-1 会调用 Close()。如果你把当前图表绑定给临时对象，生命周期结束时可能把图表关闭，使用时要控制对象生命周期。
  • 保存/读取部分有疑似不一致：Save() 写入 FixedMax、FixedMin、PointsPerBar 时使用 FileWriteDouble()，但 Load() 中读取位置使用了 FileReadDatetime()。如果后续要把它当严肃工程库使用，建议单独审查并测试这部分。
  • 汉化版未改执行逻辑：本次文件只做注释汉化，未擅自修复上述疑点，避免改变原始行为。

六、对外介绍文案

Chart.mqh 中文注释版 是一个面向 MetaTrader 开发者的图表控制基础库。它通过 CChart 类封装图表显示模式、缩放、滚动、颜色、坐标轴、事件开关、窗口尺寸、指标挂载、图表截图和参数持久化等功能，让 EA、指标和脚本开发中的图表操作更清晰、更模块化。

它不是独立交易指标，也不提供买卖信号。它更适合作为复杂 EA 面板、图表模板管理、多窗口控制、自动截图工具、指标加载管理模块的底层组件。

七、使用建议

  1. 如果用于 MT5，优先作为 Include 库文件使用。
  2. 如果用于 MT4，注意 ChartIndicatorAdd() 兼容分支直接返回 false。
  3. 如果要做商业 EA 面板，可以用它统一处理图表颜色、网格、滚动、截图和对象事件。
  4. 如果要做真正的交易指标，还需要另外编写 OnCalculate()、指标缓冲区、信号计算公式和图形输出逻辑。
