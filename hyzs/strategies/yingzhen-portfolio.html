06 · HY-ZS Portfolio

匯鹰·鹰阵组合

鹰阵组合是多策略调度与统计模块，不是直接开仓策略。它负责观察哪些策略被启用、当前总持仓如何分布、同方向敞口是否集中，以及多策略 MagicNumber 是否清晰可追踪。

组合调度敞口统计不直接开仓管理模块

适合场景

适合多个策略同时进入 Alpha 测试时使用，用来查看策略启用状态、总持仓数量、同向风险暴露和 MagicNumber 规划是否清楚。

Positioning

策略定位

鹰阵组合像一个调度台，它不替代交易信号，也不绕过鹰盾风控。

统计当前启用策略

读取七个策略开关，展示 Trend、Pullback、Range、Reversal、Scalper、Portfolio、Shield 的启用状态，帮助代理确认测试组合。

统计总持仓和方向敞口

统计账户当前持仓数量、同方向持仓数量、策略分布和可能的风险集中，作为 RiskEngine 前后的观察面板。

规划 MagicNumber

用 BaseMagicNumber 加策略编号的方式形成可追踪编号区间，方便日志、statement 和代理反馈定位问题来源。

Logic

核心逻辑

Portfolio 的核心是统计和路由，不负责产生买卖信号。

1. 策略开关扫描统计当前启用策略数量，识别是否存在测试配置冲突。
2. 持仓汇总按 MagicNumber、方向、品种和周期统计当前持仓。
3. 风险汇总接口保留总风险估算接口，用于后续评估所有策略共同风险。
4. 路由提示向总控 EA 提供可读状态，不越权发起真实交易。

Alpha 版已关注

  • 当前启用策略数量。
  • 账户总持仓数量。
  • 多策略 MagicNumber 规划说明。
  • 同方向敞口统计接口。
  • 总风险统计接口占位，便于后续接入真实风险模型。

不会做的事

  • 不直接调用 Buy 或 Sell。
  • 不替策略生成入场信号。
  • 不自动加仓、补仓或反向翻仓。
  • 不绕过 RiskEngine 修改风控结果。
  • 不为了组合收益曲线隐藏亏损策略。

禁止事项

  • 禁止把组合模块改造成隐形下单模块。
  • 禁止用组合调度绕开单策略最大持仓限制。
  • 禁止同方向策略全部开启后不统计敞口。
  • 禁止删除或隐藏策略级亏损记录。

测试重点

  • 打开不同策略组合，确认启用数量显示正确。
  • 不同 MagicNumber 的持仓能否被正确归类。
  • 多策略同向持仓时，敞口统计是否清楚。
  • 关闭 Portfolio 后，不应影响其他策略的风控开关。
  • 代理反馈中必须记录实际启用的策略组合。

风险说明

鹰阵组合本身不产生收益，它的价值是减少多策略测试时的混乱。Alpha 阶段最重要的是让每一笔交易能追溯到策略、MagicNumber、风控状态和测试配置。
